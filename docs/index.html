OpenStreetMap Editing Heatmap

What Does This Show?

Objects in OpenStreetMap are often the cumulative product of multiple edits. This visualization displays a point for every edit of a specific type (building, highway, amenity, etc.). Using a heatmap style, the density of these points then shows the relative editing activity for any given time period.

The Deleted Tiger Tags edit type shows where and when a user (or bot) deleted one of the tiger specific tags from an object (such as tiger:tlid or tiger:county)

The New Name Tags edit type shows where and when the name attribute was added to an object.

Interactivity

  1. Select the type of edit to visualize in the top left (default: Buildings)

  2. Clicking and dragging on the graph at the top will filter the map to show only the edits occuring in that time-range.

  3. Choose a specific year in the Show Years: dropdown menu to zoom the timeline to just that year.

Animation

Clicking Step will progress the selected time range at the given step value (in days, defaults to 30).

Clicking Play will step once per second until you click Stop

How this was built?

The editing history of each object was reconstructed with the OSM-Wayback utility. This utility turns historical extracts of the full OSM planet history file into a GeoJSON sequence of objects with embedded editing histories. These enriched objects are then fed into tippecanoe to create historical vector tiles.

Using tile-reduce, the historical vector tiles are processed for specific editing actions / any number of user-defined measures. The output of these scripts are individual editing records that can be turned into GeoJSON (for this visualization) or read by other analyses scripts like Python or R for further analysis.
